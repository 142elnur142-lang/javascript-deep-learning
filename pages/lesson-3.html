This Context (Контекст This)

Контекст выполнения this в различных ситуациях

Что такое this?

Ключевое слово `this` - это специальный объект, на который ссылается контекст выполнения функции. Значение `this` зависит от способа вызова функции.

1. Global Context

В глобальном контексте `this` указывает на глобальный объект (window в браузере, global в Node.js).

console.log(this); // window (в браузере) или global (в Node.js)

function globalFunction() {
    console.log(this); // window или global
}

globalFunction();

2. Method Context

Когда функция вызывается как метод объекта, `this` указывает на этот объект.

const user = {
    name: 'Alice',
    greet: function() {
        console.log(`Hello, ${this.name}`); // this = user
    }
};

user.greet(); // 'Hello, Alice'

3. Constructor Context

Когда функция вызывается с оператором `new`, `this` указывает на новый создаваемый объект.

function Person(name) {
    this.name = name; // this - новый объект
}

const person = new Person('Bob');
console.log(person.name); // 'Bob'

4. Explicit Binding - call(), apply(), bind()

call() - вызов с явным контекстом

function greet(greeting) {
    console.log(greeting + ', ' + this.name);
}

const user = { name: 'Alice' };
greet.call(user, 'Hello'); // 'Hello, Alice'

apply() - вызов с массивом аргументов

function sum(a, b, c) {
    return a + b + c + this.bonus;
}

const context = { bonus: 10 };
const result = sum.apply(context, [1, 2, 3]);
console.log(result); // 16

bind() - создание функции с закреплённым контекстом

function greet(greeting) {
    console.log(greeting + ', ' + this.name);
}

const user = { name: 'Charlie' };
const boundGreet = greet.bind(user, 'Hi');

boundGreet(); // 'Hi, Charlie'

// Также работает с частичным применением
const boundGreet2 = greet.bind(user);
boundGreet2('Hey'); // 'Hey, Charlie'

5. Arrow Functions

Стрелочные функции НЕ имеют своего `this`. Они используют `this` из окружающей области видимости.

const user = {
    name: 'Diana',
    regularMethod: function() {
        console.log(this.name); // 'Diana'
        
        const arrowFunction = () => {
            console.log(this.name); // 'Diana' (наследует this от метода)
        };
        
        arrowFunction();
    }
};

user.regularMethod();
⚡ Опасность: Никогда не используйте стрелочные функции как методы объекта, если нужен доступ к свойствам объекта через `this`.

6. Потеря контекста (Context Loss)

Частая проблема - потеря контекста при передаче метода как обработчика события.

const button = {
    name: 'Click Me',
    onClick: function() {
        console.log(this.name);
    }
};

// Проблема: потеря контекста
const handler = button.onClick;
handler(); // undefined (this = window/global)

// Решение 1: Использовать bind
document.addEventListener('click', button.onClick.bind(button));

// Решение 2: Использовать стрелочную функцию
document.addEventListener('click', () => button.onClick());

7. class Syntax

В ES6 классах методы получают правильный контекст.

class User {
    constructor(name) {
        this.name = name;
    }
    
    greet() {
        console.log(`Hello, ${this.name}`);
    }
}

const user = new User('Eve');
const greet = user.greet;
// greet(); // Ошибка: this = undefined в strict mode

// Решение: использовать стрелочные функции
class UserFixed {
    constructor(name) {
        this.name = name;
        this.greet = () => {
            console.log(`Hello, ${this.name}`);
        };
    }
}

const userFixed = new UserFixed('Frank');
const greetFixed = userFixed.greet;
greetFixed(); // 'Hello, Frank'

Практические примеры

Event Listener с сохранением контекста

class ClickCounter {
    constructor() {
        this.count = 0;
        this.button = document.querySelector('button');
        // Используем bind или стрелочную функцию
        this.button.addEventListener('click', this.handleClick.bind(this));
    }
    
    handleClick() {
        this.count++;
        console.log(`Clicked ${this.count} times`);
    }
}
