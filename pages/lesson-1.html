Closures (Замыкания)

Понимание замыканий, области видимости и лексического окружения

Что такое замыкание?

Замыкание (closure) - это функция, которая имеет доступ к переменным из своего лексического окружения, даже после того, как внешняя функция завершила свое выполнение.

Основной пример

function outer() {
    const message = 'Hello from outer';
    
    function inner() {
        console.log(message); // Доступ к переменной из outer
    }
    
    return inner;
}

const myFunction = outer();
myFunction(); // 'Hello from outer'

Как работают замыкания?

Замыкания работают благодаря лексическому окружению. Каждая функция сохраняет ссылку на переменные, которые были доступны при её создании.

Счетчик с замыканием

function createCounter() {
    let count = 0; // Приватная переменная
    
    return {
        increment: function() {
            count++;
            return count;
        },
        decrement: function() {
            count--;
            return count;
        },
        getCount: function() {
            return count;
        }
    };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.increment()); // 2
console.log(counter.decrement()); // 1
⚠️ Важно: Переменная `count` недоступна снаружи - это приватная переменная. Это основной паттерн инкапсуляции в JavaScript.

Замыкания в циклах

Частая ошибка при работе с замыканиями - неправильное использование переменных в циклах.

Проблема

const functions = [];
for (var i = 0; i < 3; i++) {
    functions.push(function() {
        console.log(i); // Проблема!
    });
}

functions[0](); // 3 (не 0!)
functions[1](); // 3 (не 1!)
functions[2](); // 3 (не 2!)

Решение 1: Использовать let

const functions = [];
for (let i = 0; i < 3; i++) {
    functions.push(function() {
        console.log(i);
    });
}

functions[0](); // 0
functions[1](); // 1
functions[2](); // 2

Решение 2: IIFE (Immediately Invoked Function Expression)

const functions = [];
for (var i = 0; i < 3; i++) {
    functions.push((function(j) {
        return function() {
            console.log(j);
        };
    })(i));
}

functions[0](); // 0
functions[1](); // 1
functions[2](); // 2

Практические применения

1. Factory функции

function makeAdder(x) {
    return function(y) {
        return x + y;
    };
}

const add5 = makeAdder(5);
const add10 = makeAdder(10);

console.log(add5(3));  // 8
console.log(add10(3)); // 13

2. Частичное применение (Partial Application)

function multiply(a, b, c) {
    return a * b * c;
}

function partial(fn, ...args) {
    return function(...nextArgs) {
        return fn(...args, ...nextArgs);
    };
}

const multiplyBy2 = partial(multiply, 2);
const multiplyBy2And3 = partial(multiplyBy2, 3);

console.log(multiplyBy2And3(4)); // 24

3. Модульный паттерн

const calculator = (function() {
    let result = 0;
    
    return {
        add: function(x) { result += x; return result; },
        subtract: function(x) { result -= x; return result; },
        multiply: function(x) { result *= x; return result; },
        getResult: function() { return result; },
        reset: function() { result = 0; }
    };
})();

console.log(calculator.add(5));      // 5
console.log(calculator.multiply(2)); // 10
console.log(calculator.subtract(3)); // 7

Производительность и утечки памяти

Замыкания сохраняют ссылки на переменные в памяти. Это может привести к утечкам памяти, если не управлять ими правильно.

⚡ Опасность: Большое количество замыканий может потребить много памяти. Всегда очищайте ненужные ссылки.

Упражнения

Упражнение 1: Создайте функцию `createMultiplier(n)`, которая возвращает функцию, умножающую аргумент на `n`.
Упражнение 2: Создайте функцию для приватного счетчика с методами: `increment()`, `decrement()`, `getValue()`.
Упражнение 3: Реализуйте функцию `memoize(fn)`, которая кэширует результаты функции используя замыкания.
